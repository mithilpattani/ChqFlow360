What's real here, and what isn't: file upload, OCR text recognition (via an in-browser engine — it actually reads your image, nothing is pre-scripted), the amount-in-words ↔ figures check, post-dated/stale date checks, duplicate cheque-number detection, and every workflow state change (submit / approve / reject / resubmit / fraud review) are computed live by this page. Signature-match is still simulated (a seed value you can edit) — that genuinely needs a trained model the real backend will provide. OCR reads printed text reasonably well but struggles with handwriting — a known limit of general-purpose OCR, not this tool; it shows you the raw recognized text and a confidence score so you can see exactly what it caught. It also only attempts structured field mapping (name/date) — it doesn't yet understand "this line is the payee" the way a cheque-trained model would. Data persists in this browser via local storage, so duplicate detection and your audit trail survive a page refresh — but not a different browser/device. Requires internet on first load (to fetch the OCR engine from a CDN, a few MB, cached afterward) — open this via the local file or GitHub Pages link, not the Artifact preview, which blocks that.
